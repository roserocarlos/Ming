Configurar Node-RED — importar el flujo
▾
⚠️ Paso obligatorio antes de importar. Si ya hubo un deploy anterior en este contenedor, las credenciales encriptadas pueden quedar corruptas. Ejecute esto primero:
2.1 — Limpiar credenciales y reiniciar Node-RED
PowerShell
docker exec ming_nodered rm -f /data/flows_cred.json docker restart ming_nodered # Espere ~10 segundos a que Node-RED vuelva a subir
2.2 — Importar el flujo nodered-flow.json

  1. Abra localhost:1880

  2. Menú ≡ (esquina superior derecha) → Import

  3. Clic en "select a file to import" → seleccione nodered-flow.json del repo

  4. En el diálogo marque replace en los dos config nodes: Mosquitto Local y [v2.0] InfluxDB Local

  5. Clic en Import selected

2.3 — Configurar el token de InfluxDB

  1. Menú ≡ → Configuration nodes → InfluxDB Local → doble clic

  2. Verifique y complete los campos:

CampoValor
URLhttp://ming_influxdb:8086
Version2.0
Tokenming-token
Organisationmyorg
Default Bucketgas_data

  1. Clic en Update → Done

  2. Clic en el botón rojo Deploy (esquina superior derecha)

  3. Los nodos MQTT deben mostrar indicador verde "connected"

El debug debe mostrar { value: 429 } cada 5 segundos — el simulador está activo.

Si aparece ❌ Error InfluxDB, el mensaje del debug indica exactamente qué falló y cómo resolverlo.

▾
3.1 — Desde la UI

  1. Abra localhost:8086 → login: admin / adminpassword

  2. Menú izquierdo → Data Explorer

  4. Filter → _measurement → gas_level

  5. Filter → _field → value

5
Instalar ESPHome y flashear el ESP8266
▾
5.1 — Instalar ESPHome
PowerShell
pip install esphome # Verificar instalación — usar siempre python -m esphome # evita bloqueos de Windows Defender python -m esphome version
Usar siempre python -m esphome en lugar de esphome directamente — evita bloqueos de Windows Defender o Control de aplicaciones.
5.2 — Editar gas-sensor.yaml

Cada grupo edita dos líneas del archivo gas-sensor.yaml:

gas-sensor.yaml — solo estas dos líneas
# Línea 1 — nombre único del AP WiFi de este grupo ssid: "MING-Gas-G1" # ← cambiar G1 por G2, G3 ... G10 # Línea 2 — IP del PC donde corre Docker broker: "192.168.0.252" # ← resultado de ipconfig
PowerShell — obtener IP del PC
ipconfig # Buscar: Adaptador LAN inalámbrica Wi-Fi → Dirección IPv4
5.3 — DIP switches — posición FLASH

Configurar antes de conectar el USB para cargar el firmware:

SW1SW2SW3SW4SW5SW6SW7SW8
OFF OFF OFF OFF ON ON ON —
5.4 — Primera carga — web installer (Chrome/Edge)

  1. Abra web.esphome.io en Chrome o Edge — no Firefox

  2. Clic en Connect → seleccione CH340 (COMx)

  3. Clic en Prepare for first use — instala firmware base ESPHome en el ESP

  4. Clic en Configure WiFi — el ESP queda en la red listo para OTA

5.5 — Carga por consola (COM directo — más confiable)
PowerShell
# Con switches en FLASH y USB conectado cd Ming python -m esphome run gas-sensor.yaml # Si hay varios puertos COM, especificar el correcto python -m esphome run gas-sensor.yaml --device COM9 # Ver logs del ESP en tiempo real por consola python -m esphome logs gas-sensor.yaml --device COM9
5.6 — DIP switches — posición RUN (después de flashear)

Solo cambia SW7 a OFF — el ESP arranca en modo operación normal:

SW1SW2SW3SW4SW5SW6SW7SW8
OFF OFF OFF OFF ON ON OFF —
Con SW7-OFF el ESP arranca, conecta al WiFi y publica en MQTT automáticamente cada 5 segundos.
6
Conectar sensor MQ-135 y verificar datos reales
▾
6.1 — Conexión del sensor a la placa
Pin MQ-135Pin placaCable
VCC5V del headerRojo
GNDGNDNegro
AOA0 (TOUT ESP8266) via divisor de voltajeVerde
DONo conectar—
⚠️ Divisor de voltaje obligatorio en AO.
El MQ-135 entrega hasta 5V en AO pero el TOUT del ESP8266 solo acepta máximo 1V.
Sin divisor puede dañar el ADC del ESP permanentemente.

R1 = 10kΩ entre AO del sensor y pin A0 de la placa
R2 = 2.2kΩ entre pin A0 y GND
Resultado: 5V → 0.9V ✅ dentro del rango seguro
6.2 — Configurar WiFi por portal captivo

  1. Encender el ESP → levanta AP automáticamente: MING-Gas-G1 (según número de grupo)

  2. Conectarse al AP desde celular o PC → contraseña: ming12345

  3. Abrir 192.168.4.1 en el navegador → ingresar red WiFi y contraseña → Guardar

  4. El ESP se conecta y recuerda la red — próximos arranques son automáticos

Si la red guardada no está disponible, el ESP vuelve a levantar el AP después de 30 segundos de intentos fallidos.
6.3 — Actualizar firmware por OTA (sin USB)

Una vez el ESP está en WiFi, todas las actualizaciones siguientes son por red:

PowerShell
# Subir cambios al ESP por WiFi (sin tocar switches ni USB) python -m esphome run gas-sensor.yaml --device 192.168.0.65 # Reemplazar 192.168.0.65 por la IP real del ESP # Ver IP en los logs o en la lista de dispositivos del router
6.4 — Apagar el simulador en Node-RED

  1. Abra localhost:1880

  2. Clic en el cuadrado azul del nodo "Trigger cada 5s" para pausarlo

  3. Clic en Deploy — ahora solo llegan datos del sensor físico

6.5 — Verificar datos reales en el debug
Debug Node-RED — datos del sensor físico
✅ Payload → InfluxDB : msg.payload : Object { value: 861 } ✅ Payload → InfluxDB : msg.payload : Object { value: 743 }
Valores altos (700–1023) son normales sin el divisor de voltaje instalado. Con el divisor R1/R2 los valores quedan en rango 200–700 según la concentración de gas en el ambiente.

Solución de problemas
▾
Node-RED no escribe en InfluxDB
PowerShell
# 1. Limpiar credenciales corruptas docker exec ming_nodered rm -f /data/flows_cred.json docker restart ming_nodered # 2. Esperar ~10 segundos, luego en Node-RED: # Configuration nodes → InfluxDB Local → Token: ming-token → Update → Deploy
ESP8266 no publica en MQTT — broker no alcanzable
PowerShell
# Verificar IP del PC en la red actual ipconfig # Buscar: Adaptador LAN inalámbrica Wi-Fi → Dirección IPv4 # Actualizar broker: en gas-sensor.yaml con esa IP # Subir de nuevo por OTA o COM
ESPHome bloqueado por Windows Defender
PowerShell
# Usar python -m en lugar del ejecutable directo python -m esphome version python -m esphome run gas-sensor.yaml --device COM9
OTA falla — mDNS no resuelve en Windows
PowerShell
# Usar IP directa en lugar del nombre .local python -m esphome run gas-sensor.yaml --device 192.168.0.65 # Ver IP del ESP en los logs del serial python -m esphome logs gas-sensor.yaml --device COM9
Bucket gas_data no encontrado
PowerShell
docker exec -it ming_influxdb influx bucket create --name gas_data --org myorg --token ming-token --retention 0
Grafana muestra "No data"

  1. Verificar que Node-RED tiene Deploy activo y nodos MQTT dicen "connected"

  2. Cambiar rango de tiempo del dashboard a Last 1 hour para incluir datos anteriores

  3. Verificar que el datasource usa Flux y URL http://ming_influxdb:8086 — no localhost

Verificar nombres exactos de los contenedores
PowerShell
docker ps --format "table {{.Names}}\t{{.Status}}" # Deben aparecer: # ming_influxdb · ming_mosquitto · ming_nodered · ming_grafana
