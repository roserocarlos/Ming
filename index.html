2
Configurar Node-RED — importar el flujo
▾
⚠️ Paso obligatorio antes de importar. Si ya hubo un deploy anterior en este contenedor, las credenciales encriptadas pueden quedar corruptas. Ejecute esto primero:
2.1 — Limpiar credenciales y reiniciar Node-RED
PowerShell
docker exec ming_nodered rm -f /data/flows_cred.json docker restart ming_nodered # Espere ~10 segundos a que Node-RED vuelva a subir
2.2 — Importar el flujo nodered-flow.json

  1. Abra localhost:1880

  2. Menú ≡ (esquina superior derecha) → Import

  3. Clic en "select a file to import" → seleccione nodered-flow.json del repo

  4. En el diálogo que aparece, marque replace en los dos config nodes:
    Mosquitto Local y [v2.0] InfluxDB Local

  5. Clic en Import selected

2.3 — Configurar el token de InfluxDB

  1. Menú ≡ → Configuration nodes → InfluxDB Local → doble clic

  2. Verifique y complete los campos:

CampoValor
URLhttp://ming_influxdb:8086
Version2.0
Tokenming-token
Organisationmyorg
Default Bucketgas_data

  1. Clic en Update → Done

  2. Clic en el botón rojo Deploy (esquina superior derecha)

  3. Los nodos MQTT deben mostrar indicador verde "connected"

El flujo arranca automáticamente y publica un valor de gas simulado cada 5 segundos en el tópico sensor/gas, escribiendo en InfluxDB.
Verificar en el panel de debug

En Node-RED, abra el panel de debug (ícono 🐛 a la derecha). Debe ver mensajes como:

Debug Node-RED
✅ Payload → InfluxDB : msg.payload : Object { value: 429 } ✅ Payload → InfluxDB : msg.payload : Object { value: 387 }

Si aparece ❌ Error InfluxDB, el mensaje del debug indica exactamente qué falló y cómo resolverlo.

3
Verificar datos en InfluxDB
▾
3.1 — Desde la UI

  1. Abra localhost:8086 → login: admin / adminpassword

  2. Menú izquierdo → Data Explorer

  3. FROM → seleccione gas_data

  4. Filter → _measurement → gas_level

  5. Filter → _field → value

  6. Rango de tiempo → Last 5 minutes → clic en Submit

Debe aparecer la gráfica con valores entre 250 y 650 cada 5 segundos.
3.2 — Desde consola (verificación rápida)
PowerShell
# Escribir un punto de prueba manualmente docker exec -it ming_influxdb influx write --bucket gas_data --org myorg --token ming-token "gas_level value=888" # Si no da error = InfluxDB funcionando correctamente
Si el bucket gas_data no existe
PowerShell
docker exec -it ming_influxdb influx bucket create --name gas_data --org myorg --token ming-token --retention 0
4
Configurar Grafana — datasource InfluxDB
▾

  1. Abra localhost:3000 → login: admin / admin

  2. Menú lateral → Connections → Data sources → Add data source → elija InfluxDB

  3. Configure exactamente estos campos:

CampoValor
Query LanguageFlux ← importante, no InfluxQL
URLhttp://ming_influxdb:8086
Organisationmyorg
Tokenming-token
Default Bucketgas_data
Query Language debe ser Flux. Con InfluxQL no aparece el campo Token y la conexión fallará.

  1. Clic en Save & Test

  2. Debe aparecer: ✅ datasource is working — 3 buckets found

5
Importar el dashboard en Grafana
▾

  1. Menú lateral → Dashboards → Import

  2. Clic en Upload dashboard JSON file → seleccione grafana-dashboard.json del repo

  3. En el selector DS_INFLUXDB → elija el datasource InfluxDB que acaba de crear

  4. Clic en Import

El dashboard MING — Monitor Sensor Gas MQ-135 queda disponible con auto-refresh de 5 segundos.

Solución de problemas
▾
Node-RED no escribe en InfluxDB
PowerShell
# 1. Limpiar credenciales corruptas docker exec ming_nodered rm -f /data/flows_cred.json docker restart ming_nodered # 2. Esperar ~10 segundos, luego en Node-RED: # Configuration nodes → InfluxDB Local → Token: ming-token → Update → Deploy
Bucket gas_data no encontrado
PowerShell
docker exec -it ming_influxdb influx bucket create --name gas_data --org myorg --token ming-token --retention 0
Grafana muestra "No data"

  1. Verifique que Node-RED tiene Deploy activo y los nodos MQTT dicen "connected"

  2. Cambie el rango de tiempo del dashboard a Last 1 hour para incluir datos anteriores

  3. Verifique que el datasource usa Flux y URL http://ming_influxdb:8086 (nombre del contenedor, no localhost)

Verificar nombres exactos de los contenedores
PowerShell
docker ps --format "table {{.Names}}\t{{.Status}}" # Deben aparecer: # ming_influxdb · ming_mosquitto · ming_nodered · ming_grafana
Comandos de mantenimiento
PowerShell
# Ver logs en tiempo real docker compose logs -f nodered # Reiniciar un servicio docker compose restart grafana # Detener el stack (conserva datos) docker compose down # ⚠️ Reset completo — elimina todos los datos docker compose down -v
