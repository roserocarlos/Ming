Levantar el stack Docker
▾

1.1 — Requisitos previos

Antes de empezar necesitas tener instalado Docker Desktop para Windows. Si no lo tienes:

  1. Descárgalo de docker.com/products/docker-desktop
  2. Instálalo con todas las opciones por defecto.
  3. Ábrelo y espera a que el ícono de la ballena en la barra de tareas quede quieto (sin animación).
ℹ️ En Docker Desktop → Settings → Resources asigna mínimo 2 GB de RAM.

1.2 — Crear las carpetas del proyecto

Abre PowerShell (tecla Windows → escribe "PowerShell" → Enter) y ejecuta:

Windows PowerShell
# Crear la estructura de carpetas del proyecto
PS C:\Users\Tu> mkdir ming-stack\mosquitto\config
PS C:\Users\Tu> mkdir ming-stack\mosquitto\data
PS C:\Users\Tu> mkdir ming-stack\mosquitto\log

Luego copia todos los archivos entregados dentro de la carpeta ming-stack\. La estructura debe quedar así:

Estructura de archivos
ming-stack/ ├── docker-compose.yml ├── gas-sensor.yaml ← ESPHome ├── nodered-flow.json ← Flujo Node-RED └── mosquitto/ └── config/ └── mosquitto.conf

2
Obtener la IP del host (para el ESP8266)
▾
⚠️ No uses localhost desde el ESP8266. Ese nombre solo funciona dentro del mismo equipo. El ESP necesita la IP real de tu PC en la red local.

Cómo obtener tu IP

Windows PowerShell
PS C:\Users\Tu> ipconfig

Busca en el resultado la sección de tu adaptador activo (el que usas para conectarte a internet). Verás algo así:

Resultado de ipconfig
Adaptador de red inalámbrica Wi-Fi:    ...    Dirección IPv4. . . . . . : 192.168.1.100 ← Esta es tu IP    Máscara de subred . . . . : 255.255.255.0    Puerta de enlace . . . . : 192.168.1.1
💡 Anota esa IP — la usarás en el siguiente paso dentro del archivo gas-sensor.yaml.
3
Flashear el ESP8266 con ESPHome Dashboard
▾
🚀 La forma más fácil: usamos el ESPHome Dashboard — una interfaz web que corre en Docker, sin instalar nada en Python. Solo copiar, pegar y hacer clic.

3.1 — Agregar ESPHome Dashboard al docker-compose

Abre el archivo docker-compose.yml con el Bloc de notas y agrega este bloque al final de la sección services:, antes de la línea volumes::

docker-compose.yml   esphome:
    image: ghcr.io/esphome/esphome
    container_name: ming_esphome
    restart: unless-stopped
    ports:
      - "6052:6052"
    volumes:
      - ./esphome_config:/config
    network_mode: host  # Necesario para detectar el ESP por USB en Windows
⚠️ Nota sobre USB en Windows: Docker Desktop no puede acceder directamente al puerto USB/COM del ESP. Para la primera carga por USB hay que usar el método alternativo de la línea de comandos (ver sección 3-B más abajo). Una vez flasheado, todas las actualizaciones se hacen por WiFi (OTA) desde el Dashboard sin este límite.

3.2 — Crear la carpeta de configuración y copiar el YAML

Windows PowerShell
PS ming-stack> mkdir esphome_config
PS ming-stack> copy gas-sensor.yaml esphome_config\

3.3 — Editar el YAML: solo 3 líneas a cambiar

Abre esphome_config\gas-sensor.yaml con el Bloc de notas y cambia únicamente esto:

gas-sensor.yaml wifi:
  ssid: "NOMBRE_DE_TU_RED_WIFI"   # ← Cambia esto
  password: "TU_CONTRASEÑA_WIFI"    # ← Cambia esto

mqtt:
  broker: "192.168.1.100"         # ← IP de tu PC (Paso 2)

Guarda el archivo.

3-A — Primera carga: método pip (más directo para USB)

Para la carga inicial por USB, instala ESPHome en Python e instala el firmware:

Windows PowerShell — Instalar ESPHome
# Paso 1: Verificar que tienes Python instalado
PS> python --version

# Paso 2: Instalar ESPHome
PS> pip install esphome

# Paso 3: Desde la carpeta ming-stack, flashear por USB
PS ming-stack> esphome run esphome_config\gas-sensor.yaml
ℹ️ ESPHome te mostrará una lista de puertos COM disponibles. Selecciona el que corresponde al NodeMCU (aparece como USB-SERIAL o Silicon Labs CP210x en el Administrador de dispositivos). Si Python no está instalado, descárgalo desde python.org — marca la opción "Add Python to PATH" durante la instalación.

3-B — Actualizaciones posteriores: OTA desde el Dashboard

Una vez que el ESP está conectado al WiFi, todas las actualizaciones siguientes se hacen así:

Reinicia el stack Docker para incluir el servicio ESPHome:

PowerShell
PS ming-stack> docker compose up -d

Abre el Dashboard de ESPHome en el navegador:

→ http://localhost:6052

Verás la tarjeta de tu dispositivo gas-sensor. Haz clic en INSTALL → selecciona Wirelessly (OTA).

ESPHome compila el firmware y lo envía por WiFi. Puedes ver los logs en tiempo real desde el mismo Dashboard con el botón LOGS.

3.4 — Verificar que el ESP publica datos

Con el ESP corriendo, verifica que llegan mensajes MQTT:

Windows PowerShell — Escuchar MQTT
PS> docker exec -it ming_mosquitto mosquitto_sub -h localhost -t "sensor/gas" -v

# Cada 10 segundos deberías ver algo como: sensor/gas 542 sensor/gas 538 sensor/gas 551

Configurar Node-RED (MQTT → InfluxDB)
▾

4.1 — Instalar el nodo InfluxDB

Abre Node-RED en el navegador: localhost:1880

Haz clic en el menú ☰ (esquina superior derecha) → Manage palette → pestaña Install

Busca: node-red-contrib-influxdb

Haz clic en Install y espera a que aparezca el mensaje de éxito.

4.2 — Importar el flujo listo

Menú ☰ → Import

Haz clic en "select a file to import" → selecciona el archivo nodered-flow.json

Clic en Import. Aparecerán 3 nodos en el canvas.

4.3 — Configurar el nodo InfluxDB

Doble clic sobre el nodo morado "Escribir en gas_data"

Junto a "Server", haz clic en el lápiz ✏️ para editar el servidor

Verifica que estos valores estén exactamente así:

CampoValor
Version2.0
URLhttp://ming_influxdb:8086
Tokenming-token
Organisationmyorg
Bucketgas_data

Clic en Update → Done

Clic en el botón rojo Deploy (esquina superior derecha). El nodo MQTT mostrará el indicador verde "connected".

5
Crear dashboard en Grafana
▾

5.1 — Agregar InfluxDB como fuente de datos

Abre localhost:3000 → login: admin / admin

Menú lateral → Connections → Data sources → Add data source → elige InfluxDB

Configura exactamente:

CampoValor
Query LanguageFlux
URLhttp://ming_influxdb:8086
Organisationmyorg
Tokenming-token
Default Bucketgas_data

Clic en Save & Test → debe aparecer ✅ "datasource is working"

5.2 — Crear el dashboard y el panel

Menú lateral → Dashboards → New → New Dashboard → Add visualization

Selecciona el datasource InfluxDB que acabas de crear

En el editor de consulta (Flux), pega esta query:

Flux from(bucket: "gas_data")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "gas_level")
  |> filter(fn: (r) => r._field == "value")
  |> aggregateWindow(every: 10s, fn: mean, createEmpty: false)
  |> yield(name: "mean")

Panel derecho → Title: Nivel de Gas MQ-135 → clic en Apply

Guarda el dashboard: ícono 💾 → nombre Gas Sensor

En la esquina superior derecha → selector de tiempo → Last 1 hour + Auto-refresh: 10s
